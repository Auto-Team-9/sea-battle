## Description of my work in the project

В рамках проекта наша команда разрабатывала обучающее веб-приложение с элементами геймификации. Основная идея — изучение программирования через игру, вдохновлённую «Морским боем»: пользователь отвечает на вопросы по выбранной теме, а правильные ответы позволяют атаковать корабли противника.

### Planning and architecture

На начальном этапе я участвовал в обсуждении концепции проекта и разработке структуры приложения. Совместно с командой мы провели несколько брейнштормов и сформировали основную механику игры, пользовательские сценарии и список основных экранов.

Также мы приняли ряд технических решений. В качестве backend-решения был выбран **Firebase**, так как он позволял быстро реализовать аутентификацию и хранение данных без разработки собственного сервера. Для базы данных мы выбрали **Cloud Firestore**, поскольку для нашего проекта важнее удобная работа с выборками данных.

---

### Technologies used

Фронтенд приложения разрабатывался с использованием:

- **React**
- **TypeScript**
- **Tailwind CSS**
- **Firebase (Auth + Firestore)**

Для стилизации интерфейса я интегрировал библиотеку **DoodleCSS**, чтобы реализовать рукописный стиль интерфейса. Поскольку библиотека конфликтовала с Tailwind, мне пришлось вендорить её и адаптировать стили. Также я интегрировал **vite-plugin-svgr** для удобного управления svg элементами.

---

### Implementation of authorization

Одной из моих основных задач была реализация системы авторизации пользователей.

Я:

- настроил **Firebase Authentication**;
- реализовал вход и регистрацию пользователей;
- подключил **OAuth-авторизацию через Google и GitHub**;
- реализовал функцию восстановления пароля;
- реализовал **protected и public routes**.

Кроме того, я разработал несколько переиспользуемых компонентов интерфейса: кнопку, инпут, систему уведомлений и компонент загрузки.

---

### Main Screen Development

После завершения работы над авторизацией я приступил к разработке **главного экрана приложения (Main Screen)**.

Я реализовал:

- header приложения с отображением аватара и имени пользователя;
- progress bar уровня и отображение ранга;
- переключение темы интерфейса;
- приветственное сообщение в зависимости от ранга;
- блок с подсказками и рекомендациями;
- систему выбора темы и уровня;
- отображение карты темы с пройденными и доступными уровнями.

Одной из сложных задач была реализация **карты уровней**. Я разработал структуру данных для хранения расположения уровней по темам и реализовал динамическое отображение карты.
